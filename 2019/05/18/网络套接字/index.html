网络编程套接字

认识端口号

端口号是传输层协议的内容。

端口号是一个2字节 16位的整数；

端口号用来标识一个进程，告诉操作系统，当前的这个数据要交给哪一个进程来处理。

IP地址 + 端口号 能够标识网络上的某一台主机的某一个进程。IP地址 + 端口号 == 套接字。

一个端口号只能被一个进程占用。

端口号 和 进程ID

不一定每个进程都有端口号，但是每个进程都有进程ID。

  • ​ 传输层协议
  • ​ 无连接
  • ​ 不可靠传输
  • ​ 面向数据流

网络字节序

先发出的数据是低地址，后发出的数据是高地址。

网络数据流都是采用大端字节序（低地址高字节），如果需要转换，再进行转换就行了，这是标准。

#include<arpa/inet.h>

uint32_t htonl(uint32_t hostlong);

uint16_t htons(uint16_t hostshort);

uint32_t ntotl(uint32_t netlong);

uint16_t ntots(uint16_t netshort);

h表示host，n表示network，l表示32位长整数，s表示16位短整数。

htonl表示将32位的长整数从主机字节序转换为32位的网络字节序。

主机是小端字节序，将参数进行大小端转换。主机是大端字节序，不需要改变。

地址转换函数

字符串转in_addr的函数：
int inet_aton(const char *strptr, struct in_addr *addrptr);
in_addr_t inet_addr(const char *strptr);
int inet_pton(int family, const char *strptr, void *addrptr);
in_addr转字符串的函数：
char *inet_ntoa(struct in_addr inaddr);
const char *inet_ntop(int family, const void *addrptr, char *strptr, size_t len);

注：inet_ntoa函数，是把这个返回结果放到了静态存储区，这个时候不需要手动释放。

inet_ntoa把结果放到自己内部的一个静态存储区，这样第二次调用时的结果会覆盖掉上一次的结果。

如果多个线程调用inet_ntoa，是会出现异常情况的。inet_ntoa不是线程安全函数。多线程下推荐inet_ntop，提供一个缓冲区保存结果，可以规避线程安全问题。

多进程服务器比较浪费内存资源，时间比较慢；当客户量足够多时，会导致CPU进行进程间切换调用的压力增大，进而导致性能的下降。

#define INADDR_ANY ((unsigned long int) 0x00000000) //表示本地的任意IP
